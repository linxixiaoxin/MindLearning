# 如何不被看起来很有道理的话带偏

**核心问题**：为什么我们明明不笨，却仍然会被强表达、故事、数据和立场轻易带偏？

## 建议阅读路径

1. 先从 学会提问（原书第10版） 进入，确认这条问题的整本书主链。
2. 再依次调用 结论、理由、价值观假设、证据质量。
3. 接着补上 相关和因果、重要遗漏、替代原因，防止被数据或单一解释带偏。
4. 最后回到 学会提问-从强表达到独立判断-最小回流样板，看这条问题链如何进入真实内容。

## 主问题链

1. 为什么强表达会比强论证更快占住注意力？
2. 为什么判断一个观点时，第一步不是表态，而是先拆结论和理由？
3. 为什么同一事实会因为不同假设和价值排序，导出完全不同的结论？
4. 为什么看起来最客观的数据，也可能在相关、因果和证据质量上误导人？
5. 为什么很多说法即使结构完整，仍然会因为重要遗漏和替代原因而失真？
6. 怎样把“先看热闹”改成“先拆结构”，最后形成可修正的独立判断？

## 用于调用的核心判断

- `强表达` 不等于 `强论证`。
- 真正的批判性思维不是先反驳，而是先拆论证结构。
- 同一事实之所以能导出相反结论，往往不是事实不同，而是前提和价值排序不同。
- 数据会增加说服感，但不会自动增加可靠性。
- 更好的独立判断不是更快站队，而是更慢地下结论、更清楚地知道依据来自哪里。

## 这张问题页回答什么

1. 为什么我们总在“看起来有道理”和“真的有依据”之间混淆？
2. 面对观点、文章、短视频和群聊转发时，最值得先问哪些问题？
3. 结论、理由、假设、证据、遗漏之间到底是什么顺序关系？
4. 为什么“有数据”“有案例”“说得很强”都还不够？
5. 什么叫更负责、也更可修正的独立判断？

## 常见误判

- 以为独立思考就是谁都不信。
- 以为批判性思维就是先挑错、先抬杠。
- 以为有数据、有案例就已经更可信。
- 以为观点越鲜明、情绪越强，结论就越站得住。

## 先看什么，再看什么

1. 学会提问（原书第10版）：先确认整条提问链的母骨架。
2. 结论：先抓住别人到底想让你接受什么。
3. 理由：再看结论到底靠什么站住。
4. 价值观假设 + 描述性假设：把没说出来的前提翻出来。
5. 证据质量：检查证据是否可信、相关、充分。
6. 相关和因果：防止被“看起来客观”的数字偷换。
7. 重要遗漏 + 替代原因：把被省略和被遮住的部分补回来。
