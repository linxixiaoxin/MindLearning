# 核心假设与验证计划

> 创建时间：2026-04-29  
> 最近更新：2026-04-29  
> 当前状态：v0.1 验证计划  
> 关联产品：思想伙伴选配器

## 1. 验证主问题

思想伙伴选配器当前最重要的验证问题是：

> 用户是否愿意从真实困境进入知识站，并把作者、书和方法理解成一组可以陪自己看问题、改表达、试行动的思想伙伴？

## 2. 核心假设

| 编号 | 假设 | 验证信号 | 风险 |
| --- | --- | --- | --- |
| H1 | 真实困境比书名入口更容易触发用户进入 | 用户从小红书或首页点击工具入口 | 困境文案太泛，用户觉得不像自己 |
| H2 | 困境重述能带来被理解感 | 用户反馈“说中了”“确实是这样” | 重述过度解释或太像鸡汤 |
| H3 | 思想伙伴组合能被理解 | 用户能说出为什么这些作者适合当前问题 | 作者过多或角色名抽象 |
| H4 | 结果卡有保存价值 | 用户截图、保存图片、转发或收藏 | 图片导出体验不好，信息太满 |
| H5 | 最小行动能降低行动门槛 | 用户愿意尝试并回填行动后反馈 | 行动建议太虚或太难 |
| H6 | 继续探索路径能带动站内深访 | 用户点击专题页、书籍节点或 N 卡 | 路径不可点击或关联不精准 |
| H7 | 行动后洞察卡能形成二次价值 | 用户愿意输入反馈并保存洞察卡 | 用户不知道该写什么，AI/规则输出不稳定 |

## 3. MVP 验证样本

第一轮目标不是大流量，而是获得 5-20 条有效反馈。

有效反馈包括：

- 用户选择了哪个困境。
- 用户是否觉得困境重述准确。
- 用户是否理解推荐组合。
- 用户是否愿意保存结果。
- 用户是否尝试了最小行动。
- 用户对洞察卡或继续探索的评价。

## 4. 关键指标

| 指标 | 采集方式 | v0.1 目标 |
| --- | --- | --- |
| 工具入口点击 | 手动观察、日志或部署平台数据 | 有真实访问即可 |
| 完成一次匹配 | 页面行为或用户反馈 | 首轮 5-20 次有效完成 |
| 保存图片意愿 | 用户截图/保存反馈 | 至少 3 条正向信号 |
| 推荐解释可信度 | 用户文字反馈 | 至少 5 条可判断反馈 |
| 行动后回填 | 洞察卡输入或用户转述 | 至少 3 条真实回馈 |
| 继续探索点击 | 站内链接或用户反馈 | v0.2 再正式计量 |

## 5. 验证方法

### 5.1 内部可用性检查

先由项目方自测：

- 8 个典型困境是否都能生成结果。
- 轻量参数改变后，结果是否有可感知变化。
- 结果卡文案是否可信、克制、可读。
- 保存图片是否在桌面和移动端都可用。
- 洞察卡生成失败时是否有可理解提示。

### 5.2 小样本用户反馈

找 5-20 个真实用户或熟人样本，让他们完成：

1. 从入口进入工具。
2. 选择一个最接近自己的困境。
3. 查看结果卡。
4. 保存或不保存结果。
5. 尝试写一段行动后反馈。
6. 回答 4 个问题：
   - 哪句话最像你？
   - 哪个推荐最可信或最不可信？
   - 你愿意保存这张卡吗？
   - 你希望下一步继续看什么？

### 5.3 内容承接测试

选择 2-3 篇小红书笔记，在结尾引导到具体工具或专题：

- 关系沟通内容 -> 关系类困境。
- 自我/无聊/刷手机内容 -> 观看、内在、行动类困境。
- 组织/管理内容 -> 社会、系统、行动类困境。

观察用户是否能从笔记语境自然进入工具。

## 6. 反馈归档字段

建议每条反馈记录这些字段：

| 字段 | 说明 |
| --- | --- |
| feedback_id | 反馈编号 |
| date | 收集日期 |
| source | 来源：内部测试 / 小红书 / 朋友 / 其他 |
| scenario_id | 所选困境编号 |
| raw_user_text | 用户原始反馈 |
| understood_signal | 是否有被理解感 |
| credibility_signal | 是否认为推荐可信 |
| save_signal | 是否保存/截图/转发 |
| action_signal | 是否愿意行动 |
| next_interest | 用户想继续看的内容 |
| product_opportunity | 可转化的产品机会 |

## 7. 决策门槛

| 决策 | 进入条件 |
| --- | --- |
| 扩展更多困境 | 至少 5 条反馈指出现有困境不够覆盖 |
| 做可点击继续探索 | 至少 3 条反馈表达想继续看相关内容 |
| 做正式 AI 洞察 Prompt | 至少 3 条真实行动后回馈样本 |
| 做历史保存 | 多个用户明确表达想回看旧结果 |
| 做个人思想地图 | 用户主动表达想长期跟踪自己的模式 |

## 8. 近期验证动作

1. 完成浏览器与移动端图片导出检查。
2. 用 8 个困境各生成一次结果，记录文案问题。
3. 找 5 个样本用户做第一轮完整体验。
4. 把反馈写入 `../02_用户研究与反馈/12_用户反馈池.md`。
5. 根据反馈决定 v0.2 是否优先做“继续探索可点击”或“AI 洞察 Prompt”。
