# 数据结构与配置字典

> 创建时间：2026-04-29  
> 最近更新：2026-04-29  
> 当前状态：v0.1 字段基线稿  
> 关联代码：`web/src/data/thoughtPartnerData.js`、`web/public/registry/books.json`

## 1. 维护原则

当前阶段优先使用前端静态配置和 JSON 文件，避免过早建设后台。

字段设计原则：

- 能解释推荐逻辑。
- 能支持保存图片和分享卡。
- 能支持后续接专题页、书籍节点和 N/K 卡。
- 能支持 AI / Codex 交接。
- 不收集不必要的用户身份信息。

## 2. 困境配置

建议结构：

```js
{
  id: "S01",
  title: "刷完小红书更讨厌自己的生活",
  primaryAxis: "seeing",
  secondaryAxes: ["social", "inner"],
  restatement: "...",
  lens: "...",
  defaultAction: "...",
  explorePaths: []
}
```

字段说明：

| 字段 | 类型 | 说明 | 必填 |
| --- | --- | --- | --- |
| `id` | string | 困境编号 | 是 |
| `title` | string | 用户可见困境标题 | 是 |
| `primaryAxis` | string | 主能力轴 | 是 |
| `secondaryAxes` | array | 辅助能力轴 | 是 |
| `restatement` | string | 困境重述 | 是 |
| `lens` | string | 学科母题镜头 | 建议 |
| `defaultAction` | string | 默认最小行动 | 是 |
| `explorePaths` | array | 继续探索路径 | 建议 |

## 4. 思想伙伴配置

建议结构：

```js
{
  id: "bachelard",
  name: "巴什拉",
  source: "空间的诗学",
  axes: ["seeing", "inner"],
  roles: ["mainLens", "expressionPartner"],
  tones: ["poetic"],
  fitReason: "...",
  avoidWhen: "...",
  links: []
}
```

字段说明：

| 字段 | 类型 | 说明 | 必填 |
| --- | --- | --- | --- |
| `id` | string | 内部标识 | 是 |
| `name` | string | 作者、思想或方法名 | 是 |
| `source` | string | 来源书籍或方法 | 建议 |
| `axes` | array | 适配能力轴 | 是 |
| `roles` | array | 可担任的伙伴角色 | 是 |
| `tones` | array | 表达气质标签 | 建议 |
| `fitReason` | string | 推荐理由 | 建议 |
| `avoidWhen` | string | 不适用边界 | 可选 |
| `links` | array | 相关专题/书籍/N卡 | 建议 |

## 5. 伙伴角色枚举

| id | 中文名 |
| --- | --- |
| `mainLens` | 主镜头 |

## 8. 继续探索路径

建议结构：

```js
{
  type: "topic",
  title: "安全感先于技巧",
  href: "/topics/safety-before-skill",
  reason: "继续看关系沟通中的安全感问题"
}
```

类型建议：

| type | 说明 |
| --- | --- |
| `topic` | 专题页 |
| `book` | 书籍首页 |
| `note` | 书籍节点或阅读器节点 |
| `n_card` | N 卡 |
| `k_theme` | K 主题 |
| `external` | 外部表单或链接 |

## 9. Codex inbox 请求对象

建议结构：

```json
{
  "type": "thought_partner_insight_request",
  "version": "0.1",
  "createdAt": "2026-04-29T00:00:00+08:00",
  "scenario": {},
  "result": {},
  "userReflection": "",
  "expectedResponseSchema": {}
}
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| `type` | 请求类型 |
| `version` | 协议版本 |
| `createdAt` | 创建时间 |
| `scenario` | 用户所选困境 |
| `result` | 当前结果卡结构 |
| `userReflection` | 用户行动后反馈 |
| `expectedResponseSchema` | 期望 Codex 返回结构 |

| 字段 | 类型 | 说明 |

| `version` | string | 协议版本 |
| `pattern` | string | 我看见的模式 |
| `meaning` | string | 这说明什么 |
| `suggestion` | string | 新的建议 |

## 11. 数据质量检查

每次改配置时检查：

- 所有困境都有主轴和最小行动。
- 所有角色都能匹配到伙伴。
- 所有用户可见文案无内部字段名泄露。
- `explorePaths` 不出现死链。
- Codex JSON 返回字段能被页面解析。
- 保存图片使用的字段不为空。
