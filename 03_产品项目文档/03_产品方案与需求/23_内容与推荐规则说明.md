# 内容与推荐规则说明

> 创建时间：2026-04-29  
> 最近更新：2026-04-29  
> 当前状态：v0.1 规则说明稿  
> 关联数据：`web/src/data/thoughtPartnerData.js`

## 1. 规则定位

当前推荐规则不是机器学习模型，也不是权威测评。

它是一个轻量启发式匹配系统，用于验证：

- 用户是否能感知到不同困境会得到不同思想组合。
- 用户是否能理解“作者不是答案，而是功能不同的思想伙伴”。
- 用户是否愿意沿着推荐继续探索书、专题和 N/K 卡。

## 2. 输入结构

### 2.1 困境

每个困境至少包含：

- 编号：如 `S01`
- 困境标题
- 主能力轴
- 辅助能力轴
- 困境重述模板
- 学科母题镜头
- 默认最小行动
- 推荐探索路径

### 2.2 轻量参数

| 参数 | 作用 |
| --- | --- |
| 承接方式 | 影响用户想被安放、校准、推动还是表达 |
| 表达气质 | 影响作者和语言风格倾向 |
| 当前目标 | 影响结果收束到理解、表达、行动还是继续探索 |

### 2.3 作者 / 思想 / 方法

每个候选伙伴建议包含：

- 名称
- 所属书籍或方法来源
- 擅长的问题类型
- 对应能力轴
- 可担任的伙伴角色
- 表达气质标签
- 推荐理由
- 适用边界
- 可连接的专题页 / 书籍节点 / N 卡

## 3. 能力轴

| 能力轴 | 说明 | 典型问题 |
| --- | --- | --- |
| 观看 | 重新看见事物、空间、日常和注意力 | “我是不是把生活看窄了？” |
| 内在 | 理解自己的感受、欲望、防御和价值感 | “我为什么这样反应？” |
| 关系 | 理解亲密关系、沟通、边界和选择感 | “我为什么在关系里卡住？” |
| 社会 | 看见比较、规范、信息环境和外部压力 | “为什么我总觉得不够好？” |
| 叙事 | 改写旧剧本、身份故事和人生解释 | “我是不是活在旧故事里？” |
| 行动 | 把理解变成小动作和可重复练习 | “我懂了，但怎么开始？” |

能力轴不是要求用户再选一次问题，而是系统对当前困境的理解维度。

## 4. 五类思想伙伴角色

| 角色 | 功能 | 结果中的表达方式 |
| --- | --- | --- |
| 主镜头 | 提供主要观看角度 | “这次先让谁帮你照亮问题” |
| 翻译者 | 把模糊感受翻译成人话 | “它把说不清的感受翻成可理解语言” |
| 校正者 | 防止过度解释、误判或自责 | “它帮你校准方向，不把问题全归咎于自己” |
| 行动伙伴 | 给出最小行动和练习 | “它帮你把理解落到今天能做的一步” |
| 表达伙伴 | 帮用户形成可说出口的表达 | “它帮你把洞察说得更像自己” |

## 5. 匹配逻辑

### 5.1 基础匹配

1. 根据困境主轴和辅助轴筛选候选伙伴。
2. 按候选伙伴支持的角色分配到五类功能位。
3. 每个角色优先选择与当前困境和参数最匹配的伙伴。
4. 如果某角色候选不足，用默认稳定候选补足。

### 5.2 参数加权

| 参数选择 | 可能提高的权重 |
| --- | --- |
| 被安放 | 翻译者、表达伙伴 |
| 被推动 | 行动伙伴 |
| 被校准 | 校正者 |
| 诗性表达 | 观看、内在、表达相关伙伴 |
| 行动表达 | 行动、方法、习惯相关伙伴 |
| 结构表达 | 社会、系统、组织相关伙伴 |
| 找到行动 | 行动伙伴、最小行动 |
| 形成表达 | 表达伙伴、可说出口的句子 |
| 理解自己 | 内在、观看、叙事相关伙伴 |

## 6. 推荐解释规则

每次推荐至少解释三件事：

1. 这个困境为什么需要这些能力轴。
2. 五类伙伴分别承担什么功能。
3. 当前参数如何影响了这次组合。

好的解释应当像这样：

```text
这次不是要立刻解决问题，而是先把“我怎么又这样了”的自责拆开。
主镜头负责帮你重新看见困境，翻译者负责把模糊感受翻成人话，
行动伙伴只给一个足够小的动作，避免新的压力。
```

避免：

- “你就是……”
- “真正的问题是……”
- “你必须……”
- “唯一正确的方法是……”

## 7. 最小行动规则

最小行动必须满足：

- 今天可做。
- 不需要额外工具或复杂准备。
- 不承诺立刻改变人生。
- 能生成行动后反馈。

示例：

- 写下刚才最想逃开的 1 个瞬间。
- 把“我不够好”改写成“我正在和哪个标准比较”。
- 对一个关系场景只描述事实，不急着解释对错。

## 8. 继续探索路径

继续探索路径应优先连接：

1. 已有专题页。
2. 已有书籍节点。
3. 已有 N/K 卡。
4. 后续待建设的场景库。

路径不是为了堆更多内容，而是为了回答：

> 如果用户觉得这张卡有用，下一步最自然该看哪里？

## 9. 质量边界

推荐内容应避免：

- 心理诊断化。
- 过度承诺。
- 过度作者崇拜。
- 单一价值观压迫。
- 把复杂问题简化成一句行动口号。

推荐内容应保持：

- 温和。
- 可解释。
- 有边界。
- 能回到用户的真实场景。
- 能连接到已有知识资产。
