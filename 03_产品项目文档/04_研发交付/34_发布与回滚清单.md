## 1. 发布目标

确保 `book-kb-multi` 每次发布前都能确认：

- 构建通过。
- 核心路由可访问。
- 思想伙伴选配器核心流程可用。
- 保存图片和 fallback 不阻塞。
- 如出现问题，可以回滚到上一版。

## 3. 构建检查

```text

```

- `web/dist/` 成功生成。

## 7. Netlify Function 检查

如果本次涉及 AI 接口：

| 检查项 | 说明 |
| --- | --- |
| `OPENAI_API_KEY` | Netlify 环境变量是否已配置 |
| `OPENAI_MODEL` | 如有指定，确认模型名 |
| fallback | key 未配置时页面是否仍可用 |
| 错误提示 | 接口失败时不暴露敏感信息 |

## 8. 发布命令

当前 `web/package.json` 提供：

```text
npm run deploy
```

该命令等价于：

```text
npm run build && netlify deploy --prod --dir=dist
```

发布前请确认：

- 已登录 Netlify CLI。
- 当前站点 link 正确。
- 本次确实要发布 production。

如需先发 preview，应使用 Netlify CLI preview deploy，而不是直接 production。

## 10. 回滚策略

### 10.1 Netlify 回滚

如果 production 发布后出现严重问题：

1. 进入 Netlify Deploys。
2. 找到上一版稳定 deploy。
3. 选择恢复/发布上一版。
4. 记录回滚原因和影响范围。

### 10.2 Git 回滚

如果需要代码层回滚：

- 优先使用新 commit 修复或 revert 指定 commit。
- 不使用破坏性 reset。
- 回滚前确认是否有用户未提交改动。

### 10.3 功能降级

思想伙伴选配器优先保留页面可用：

- AI 接口失败时回退本地规则。
- 保存图片失败时提示截图。
- Codex bridge 不可用时保留复制 Prompt。
