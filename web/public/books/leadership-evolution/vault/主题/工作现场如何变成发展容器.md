# 工作现场如何变成发展容器

> 这是一个 K / Knowledge 主题页。 核心问题：如果发展不该只发生在培训课上，组织怎样把会议、反馈、协作和管理动作改造成持续成长的容器？

## 这篇内容在讲什么

1. 为什么组织如果只把发展交给培训，就很难跟上复杂环境？
2. 什么样的工作场，才真的会推动人发展，而不是只消耗人？
3. 反馈和会议为什么是最值得重写的组织装置？
4. 怎样把“完成工作”和“发展人”同时纳入组织的真实任务？

## 为什么这很重要

1. 成人的大部分清醒时间都在工作里，组织真正的发展策略不能绕开工作现场。
2. 工作之所以常常不带来成长，不是因为工作本身无力，而是因为会议、反馈和协作装置没有被设计成发展容器。
3. 一旦把反馈重写成[[反馈后学习]]，反馈就从“我纠正你”转成“我们一起理解和修正”。
4. 一旦把会议重写成[[发展型会议]]，会议就不只生产行动项，也开始生产新的共同理解。
5. 所以发展型组织的关键，不是再多做一套学习系统，而是让组织运行本身就携带学习。

## 可以从哪里进入

1. 从 [[工作即成长场]] 进入，先建立“工作本身就是发展场”的总前提。
2. 再看 [[反馈后学习]] 和 [[发展型会议]]，把理念落到高频组织装置。
3. 最后接到 [[转化性心智习惯]]，看怎样让组织容器和个体练习彼此加强。
