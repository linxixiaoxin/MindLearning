# 测试计划与验收用例

> 创建时间：2026-04-29  
> 最近更新：2026-04-29  
> 当前状态：v0.1 验收基线稿  
> 关联页面：`/tools/thought-partner`

## 1. 测试目标

确保思想伙伴选配器在 v0.1 阶段具备基本可用性：

- 用户能进入工具并完成一次匹配。
- 不同困境和参数能生成合理变化。
- 结果卡和洞察卡能保存图片。
- Codex 交接链路可理解、可回填。
- AI 接口不可用时有稳定 fallback。
- 桌面端和移动端无明显遮挡、裁切和重叠。

## 8. 响应式与可访问性检查

建议至少检查这些视口：

| 视口 | 说明 |
| --- | --- |
| 390x844 | 常见手机竖屏 |
| 768x1024 | 平板或窄屏 |
| 1366x768 | 常见桌面 |
| 1440x900 | 宽屏桌面 |

检查点：

- 按钮文字不溢出。
- 卡片不互相遮挡。
- 结果区不会被导航遮住。
- 保存图片按钮可点击。
- 移动端阅读顺序自然。

## 9. 构建与发布前检查

发布前至少执行：

```text
npm run build
```

如涉及本地桥接，额外执行：

```text
npm run codex:bridge
```

并手动检查：

- `/books`
- `/tools/thought-partner`
- 任一已有专题页
- 保存结果图片
- 保存洞察卡图片

## 10. 验收标准

v0.1 可以进入小样本测试的最低标准：

- P0 用例全部通过。
- 8 个困境都能生成结果。
- 结果卡 PNG 在桌面和移动端至少各通过一次。
- AI 接口不可用时 fallback 正常。
- 用户不需要理解内部技术链路，也能完成一次结果生成和保存。
