## 1. 反馈池用途

本文件用于沉淀思想伙伴选配器第一轮真实反馈，帮助判断：

- 困境入口是否贴近真实用户。
- 困境重述是否带来被理解感。
- 思想伙伴组合是否可信。
- 结果卡是否值得保存或分享。
- 最小行动是否真的降低行动门槛。
- 用户是否愿意继续探索专题页、书籍节点或 N/K 卡。

这不是普通留言收集，而是 v0.1 到 v0.2 的产品证据池。

## 2. 有效反馈标准

一条有效反馈至少包含以下 3 项：

- 用户选择或提到的困境。
- 用户对结果卡的具体反应。
- 用户对下一步的具体意愿或阻碍。

更理想的反馈包含：

- 是否觉得“被说中”。
- 哪个思想伙伴最可信或最不可信。
- 是否愿意保存图片。
- 是否尝试最小行动。
- 行动后有什么新观察。

## 5. 第一轮重点问题

收集反馈时优先问这 6 个问题：

1. 你选的困境像不像你现在遇到的问题？
2. 结果里哪句话最像你？
3. 哪个思想伙伴推荐让你觉得可信？哪个不可信？
4. 你愿意保存这张结果卡吗？为什么？
5. 最小行动你愿意试吗？哪里太难或太虚？
6. 如果继续看，你更想看相关专题、书里的节点，还是自己的长期地图？

## 6. 反馈分型

后续每条反馈可归入以下类型：

| 类型 | 说明 | 可能动作 |
| --- | --- | --- |
| 困境命中 | 用户觉得入口很像自己 | 保留并强化该困境 |
| 困境偏差 | 用户觉得问题不像自己 | 修改困境文案或新增困境 |
| 解释不足 | 用户不理解为什么推荐这些伙伴 | 补推荐解释 |
| 信息过载 | 用户觉得结果太满 | 简化结果卡或拆分层级 |
| 保存价值 | 用户愿意截图/保存/转发 | 强化图片导出和分享卡 |
| 行动阻塞 | 用户觉得最小行动仍然太难 | 重写最小行动规则 |
| 探索意愿 | 用户想继续看更多 | 优先建设可点击继续探索 |
| 长期需求 | 用户想要个人长期思想地图 | 进入 v1.0 候选 |

## 7. 从反馈到需求的转换规则

| 反馈信号 | 转换方式 |
| --- | --- |
| 3 条以上提到同一困境缺失 | 进入 `../03_产品方案与需求/24_版本需求池与优先级.md`，候选新增困境 |
| 3 条以上觉得推荐不可信 | 优先优化 `../03_产品方案与需求/23_内容与推荐规则说明.md` 和数据配置 |
| 3 条以上愿意保存 | 优先保障保存图片和分享卡体验 |
| 3 条以上想继续探索 | 优先做可点击专题页 / 书籍节点 / N 卡 |
| 3 条以上写行动后反馈 | 优先补正式 AI 洞察 Prompt |
| 多人想回看历史 | 记录为 v1.0 历史结果需求 |

## 9. 反馈安全与隐私

- 不记录真实姓名、手机号、微信号等可识别信息。
- 用户原话如涉及隐私，应改写摘要。
- 涉及关系、心理、工作冲突的内容，只保留产品判断所需信息。
- 反馈用于产品改进，不作为诊断材料。
