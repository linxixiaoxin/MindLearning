## 5. 命名与维护规范

- 文件夹使用两位数字前缀，按项目生命周期排序。
- 文件继续使用两位数字前缀，保留原有编号体系。
- 根目录只保留总索引 `README.md`，具体文档进入对应领域文件夹。
- 已经在项目根目录存在的战略和推进记录暂不迁移，避免破坏现有引用。
- 每份文档顶部建议保留：创建时间、最近更新、当前状态、关联代码或数据文件。
- 每次重要功能推进后，至少同步更新：
  - [36_研发任务与需求同步](./04_研发交付/36_研发任务与需求同步.md)
  - [24_版本需求池与优先级](./03_产品方案与需求/24_版本需求池与优先级.md)
  - [35_缺陷与技术债记录](./04_研发交付/35_缺陷与技术债记录.md)
  - [50_关键决策记录_ADR](./06_决策复盘/50_关键决策记录_ADR.md)
