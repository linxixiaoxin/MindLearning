# 为什么一团和气的团队反而更危险

**核心问题**：为什么一个看起来很少冲突、很快统一、很会顾全大局的团队，反而可能更难做出高质量判断？

## 建议阅读路径

1. 先看 [[走出心智的误区]]，确认这条问题来自本书哪条主线。
2. 再调用 [[社会疼痛]]、[[渴求共识]]、[[妥协]]、[[第三选择]]。
3. 最后回到真实会议和跨部门协作，检查团队现在到底在“止痛”，还是在“解题”。

## 主问题链

1. 为什么复杂问题天然会带来差异，而差异又总让人不舒服？
2. 为什么团队一感觉不舒服，就会急着结束分歧，而不是继续把问题谈深？
3. 为什么“各退一步”看起来成熟，结果却常常把真正有价值的信息一起压薄？
4. 如果不要[[妥协]]也不要站队，团队到底该怎么长出更高质量的方案？

## 用于调用的核心判断

- 团队缺少冲突，不一定说明成熟，也可能说明真实差异已经不敢上桌。
- 很多所谓高效共识，本质上只是快速止痛，而不是高质量解题。
- 真正成熟的团队，不是更快统一口径，而是更能承受分歧里的张力，直到长出更好的理解。

## 这张问题页回答什么

1. 为什么“和气”在复杂问题里常常是一种危险信号？
2. [[社会疼痛]]、沉默、[[妥协]]和站队之间是什么关系？
3. 怎样判断一个团队是在形成真实共识，还是在形成低质量共识？
4. 什么叫[[第三选择]]，它为什么不是折中值？

## 常见误判

- 会议上争论少，说明团队更成熟。
- 只要大家都点头，执行就一定会更顺。
- [[妥协]]比冲突高级，所以复杂问题最好也尽快折中。
- 团队一旦出现分歧，就说明关系出了问题。

## 先看什么，再看什么

1. [[社会疼痛]]：先理解人为什么会本能回避分歧。
2. [[渴求共识]]：再看“尽快一致”为什么会变成默认动作。
3. [[妥协]]：接着辨认很多折中其实只是快速止痛。
4. [[第三选择]]：最后看什么才算超越[[妥协]]的高质量新方案。
